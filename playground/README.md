# TrendAI PM Remote Kiro IDE Environments

Provisions 8 isolated, browser-accessible **Kiro IDE** environments on AWS for TrendAI Product Managers.

## What Each PM Gets

- A dedicated EC2 instance with full Linux desktop (GNOME)
- **Kiro IDE** (v0.12.224) installed and ready to launch
- **Kiro CLI** installed
- **Amazon DCV** for browser-based remote desktop access (port 8443)
- Pre-installed: Node.js 20 LTS, Git, npm
- The [kiro-pm-starter](https://github.com/awsbenson/kiro-pm-starter.git) repo cloned at `~/kiro-pm-starter`
- Publicly accessible via browser (no VPN needed)
- Individual isolated environments (no shared state)

## Architecture

```
Browser (any device)
    │
    ▼ HTTPS :8443
┌─────────────────────────────┐
│  EC2 (t3.xlarge)            │
│  ┌───────────────────────┐  │
│  │  Amazon DCV Server    │  │
│  │  (web client)         │  │
│  └──────────┬────────────┘  │
│             ▼               │
│  ┌───────────────────────┐  │
│  │  Ubuntu Desktop       │  │
│  │  + Kiro IDE           │  │
│  │  + Node.js / Git      │  │
│  │  + Starter Kit        │  │
│  └───────────────────────┘  │
└─────────────────────────────┘
```

Each PM gets their own EC2 instance with a unique public IP, username, and password.

## Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.5
- AWS CLI configured with the `aai_benlai` profile
- SSH key pair (auto-generated by the Makefile)

## Quick Start

```bash
# 1. Deploy all 8 environments
make apply

# 2. Get the URLs for each PM
make urls

# 3. Get full credentials (username + password)
make credentials

# 4. Check instance status
make status
```

## Access Instructions (for PMs)

1. Open the URL in any modern browser (Chrome, Firefox, Edge, Safari)
2. Accept the self-signed certificate warning (click "Advanced" → "Proceed")
3. Log in with your username and password
4. You'll see a full Linux desktop — double-click the **Kiro IDE** icon to launch
5. Your starter project is at `~/kiro-pm-starter`

## Commands

| Command            | Description                              |
|--------------------|------------------------------------------|
| `make apply`       | Deploy all 8 environments                |
| `make urls`        | Show access URLs                         |
| `make credentials` | Show usernames and passwords (sensitive) |
| `make status`      | Check EC2 instance states                |
| `make destroy`     | Tear down all environments               |
| `make plan`        | Preview what Terraform will create       |

## Configuration

| Variable         | Default        | Description                    |
|------------------|----------------|--------------------------------|
| `pm_count`       | 8              | Number of environments         |
| `instance_type`  | t3.xlarge      | EC2 instance type              |
| `volume_size`    | 40 GB          | EBS volume size                |
| `aws_region`     | ap-east-2      | AWS region (Taipei)            |
| `aws_profile`    | aai_benlai     | AWS CLI profile                |

Edit `terraform/variables.tf` or pass overrides:
```bash
cd terraform && terraform apply -var="pm_count=5"
```

## Cost Estimate (ap-east-2)

Per environment (24/7 operation):
- EC2 t3.xlarge: ~$140/month
- EBS 40 GB gp3: ~$4/month
- **Total per PM: ~$144/month**
- **Total for 8 PMs: ~$1,152/month**

Stop instances when not in use to reduce costs significantly.

## Tear Down

```bash
make destroy
```

## Security Notes

- Each environment uses a unique randomly-generated password
- DCV uses HTTPS (self-signed cert) on port 8443
- SSH access on port 22 is available for troubleshooting
- For production use, consider restricting SSH to known IPs
- Amazon DCV is free on EC2 instances (license validated via IAM role + S3)
