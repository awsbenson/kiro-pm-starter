# Stakeholder Communication

## One Fact Base → Six Outputs

Given the same incident, produce audience-specific versions:

1. **Customer Executive** — Business impact, ownership, timeline. No jargon.
2. **Customer Technical** — Evidence, workaround steps, investigation status.
3. **Engineering Brief** — Reproduction data, hypotheses, severity justification.
4. **Product Manager Brief** — Feature gap, segment impact, roadmap implication.
5. **Sales/Account Update** — Renewal risk, sentiment, talking points, what NOT to promise.
6. **Internal Executive** — Strategic risk, decision needed, resource ask.

## Rules
- Never copy-paste the same update to different audiences
- Each version uses the audience's language and priorities
- All versions must be factually consistent (same facts, different framing)
- Include "Last Updated" timestamp on all versions

## Quality Check
Before sending any communication:
- [ ] Facts and assumptions clearly separated?
- [ ] Business impact quantified?
- [ ] Every action item has owner + due date?
- [ ] Tone appropriate for audience?
- [ ] No internal blame or finger-pointing in customer-facing versions?
