# Project Coordination

## Project Register Entry
Every initiative must have:
- Project name and one-line description
- Owner and stakeholders
- Status (planning / active / blocked / complete)
- Dependencies on other teams
- Key milestones with dates
- Risk and mitigation
- Resource requirements

## Cross-Team Coordination Rules
- Identify overlapping work before starting new initiatives
- Flag resource conflicts early (same engineer on multiple projects)
- Weekly status must fit in 3 bullets
- Blocked items escalate after 48 hours with no progress
- Handoff between regions must include written context (no verbal-only)

## AMEA Handoff Checklist
- [ ] Written status update posted
- [ ] Open action items listed with owners
- [ ] Customer expectations for next 12 hours documented
- [ ] Escalation path defined if situation worsens
- [ ] All stakeholders notified of handoff
